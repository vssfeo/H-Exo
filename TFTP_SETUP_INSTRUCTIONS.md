# Настройка TFTP для разработки на NanoPi M4 (RK3399)

## 1. Настройка TFTP сервера (ваш ПК)

### Вариант 1: Использование встроенного TFTP сервера

1. Убедитесь, что у вас создана директория `C:\tftpboot`
2. Файл ядра `kernel_neuro.bin` должен находиться в этой директории
3. IP-адрес вашего ПК: `192.168.1.166`

### Вариант 2: Использование Tftpd64

1. Загрузите Tftpd64 с официального сайта
2. Используйте созданный файл конфигурации `Tftpd64.ini`
3. Запустите Tftpd64 и убедитесь, что он слушает порт 69

## 2. Настройка клиента (NanoPi M4 / U-Boot)

В консоли U-Boot выполните следующие команды один раз:

```
setenv ipaddr 192.168.1.10
setenv serverip 192.168.1.166
setenv bootfile kernel_neuro.bin
setenv netboot 'tftp ${loadaddr} ${bootfile}; go ${loadaddr}'
saveenv
```

После настройки вы можете загружать ядро командой:
```
run netboot
```

## 3. Автоматизация через PowerShell скрипт

Для автоматизации процесса развертывания используйте скрипт `deploy_tftp.ps1`:

```powershell
.\deploy_tftp.ps1
```

Этот скрипт выполняет следующие действия:
1. Компилирует ядро (если есть Makefile.neuro)
2. Копирует бинарный файл в директорию TFTP
3. Отправляет команду перезагрузки через UART (в разработке)

## 4. Проверка связи

Для проверки связи в U-Boot выполните:
```
ping 192.168.1.166
```

Если вы видите "host is alive", то все настроено правильно.

## 5. Использование

Теперь ваш цикл разработки выглядит так:
1. Внесите изменения в код (например, в `neuro/adaptive_scheduler.c`)
2. Запустите `deploy_tftp.ps1`
3. В консоли U-Boot выполните `run netboot`
4. Ядро загрузится по сети за 2 секунды вместо 3-5 минут с SD-карты

## 6. Устранение неполадок

Если загрузка не работает:
1. Проверьте, что TFTP сервер запущен и слушает порт 69
2. Убедитесь, что файл `kernel_neuro.bin` находится в директории `C:\tftpboot`
3. Проверьте сетевое подключение между ПК и NanoPi
4. Убедитесь, что настройки IP-адресов в U-Boot корректны
