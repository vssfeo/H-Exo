# TFTP Setup для разработки на NanoPi M4 (RK3399)

Этот документ описывает настройку TFTP окружения для ускорения цикла разработки на NanoPi M4.

## Содержание файлов

1. `deploy_kernel_tftp.ps1` - Основной скрипт для развертывания ядра через TFTP
2. `start_tftp_server.ps1` - Скрипт для запуска TFTP сервера
3. `Tftpd64.ini` - Конфигурационный файл для Tftpd64
4. `TFTP_SETUP_INSTRUCTIONS.md` - Подробные инструкции по настройке

## Быстрый старт

1. **Запустите TFTP сервер**:
   ```powershell
   .\start_tftp_server.ps1
   ```

2. **Настройте NanoPi M4** (однократно):
   В консоли U-Boot выполните:
   ```
   setenv ipaddr 192.168.1.10
   setenv serverip 192.168.1.166
   setenv bootfile kernel_neuro.bin
   setenv netboot 'tftp ${loadaddr} ${bootfile}; go ${loadaddr}'
   saveenv
   ```

3. **Разверните ядро**:
   ```powershell
   .\deploy_kernel_tftp.ps1
   ```

4. **Загрузите ядро на NanoPi**:
   В консоли U-Boot выполните:
   ```
   run netboot
   ```

## Подробные инструкции

Подробные инструкции по настройке находятся в файле `TFTP_SETUP_INSTRUCTIONS.md`.

## Преимущества TFTP

- Время загрузки ядра: 2 секунды вместо 3-5 минут
- Нет необходимости выключать плату и извлекать SD-карту
- Автоматизация процесса развертывания
- Упрощение тестирования изменений в коде

## Устранение неполадок

Если возникают проблемы:

1. Проверьте, что TFTP сервер запущен и слушает порт 69
2. Убедитесь, что файл `kernel_neuro.bin` находится в директории `C:\tftpboot`
3. Проверьте сетевое подключение между ПК и NanoPi
4. Убедитесь, что настройки IP-адресов в U-Boot корректны

## Дополнительные возможности

Скрипт `deploy_kernel_tftp.ps1` поддерживает параметры:

- `-AutoReboot` - автоматическая отправка команды загрузки через UART
- `-ComPort` - указание COM-порта для UART
- `-BaudRate` - указание скорости передачи данных через UART

Пример использования с параметрами:
```powershell
.\deploy_kernel_tftp.ps1 -AutoReboot -ComPort "COM4" -BaudRate 115200
```
